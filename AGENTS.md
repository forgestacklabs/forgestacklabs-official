## Edit Approval Workflow

When modifying files through the IDE:

* Always use the IDE's Apply/Approve edits workflow.
* Always present changes in the built-in review/diff interface before considering them accepted.
* After applying changes, provide the IDE's built-in revert/undo action when available.
* Do not substitute Git commands for review or undo if the IDE workflow supports review and revert.
* Wait for user approval before treating changes as final.
